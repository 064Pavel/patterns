# декаратор (структурный)

## определение
это структурный паттерн проектирования, который позволяет динамически добалять объектам новую функциональность

## какую проблему решает?
решает проблему добавления новой функциональности к объектам без изменения основного кода, обеспечивая гибкость комбинирования функциональности во время выполнения программы

## применение
когда нам нужно изменять динамически функциональность объекта или расширить финализированный класс
  
пример: к существуещему коду добавить логирование

## шаги реализации
1. создать абстракции для конкретного класса к которому хтим добавить функциональность
2. создать конкретный класс, который реализует данную абстракцию и преоставляет базовую функциональность
3. создать абстрактный класс-декоратор, который реализует ту же абстракцию, что и конкретный класс
4. создать конкретные классов-декараторов

## приемущества 
* реализует OCP 
* большая гибкость, чем у наследования
* позволяет добавлять функциональность на лету
* возможность отказаться от god-object в сторону множества классов декораторов

## недостатки
* усложняет структуру
* много мелких декораторов
